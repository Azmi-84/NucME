# Contributing to CriLA

Thank you for your interest in contributing! I welcome bug reports, documentation updates, code improvements, and new knowledge.

## Remember

As this is a learning project, make sure you understand what you wish to contribute.

## Use of Artificial-Intelligence (AI)

Yes, you can use artificial intelligence, but effectively. Make sure you understand what you are writing and doing using these tools. Please, do not waste my time by adding garbage to the codebase. Always keep in mind that foundational knowledge is very important, whether big tech says so. At the end of the day, you will be Google is hiring those who can write code on paper, not on a computer. At that time, your brain will help you, not your subscribed AI or coding agents. So, always wish to help others on how they can improve their knowledge by utilizing their brain.

## How Can I Contribute?

### 1. Reporting Bugs

1. Ensure the bug hasn’t been reported in the Issues Tab.
2. I personally follow the file name convention like: 01_Data_Collection. So, make sure you follow this as well.
3. For the function name, please make sure you follow the convention like: dataset_cleaning.
4. Make sure you do not write code more than 100 lines; for notebooks, that’s a different story, as it will exceed, but for raw code, if you think it is necessary for a particular section, follow the code line convention.

### 2. Submitting Code Changes

1. Fork the repository and Clone your fork.
2. Create a branch for your changes.
3. Commit with a clear message.
4. Push the branch and open a Pull Request.

## Pull Request Guidelines

1. Ensure code passes tests and updates documentation.
2. Link the Pull Request to the relevant issue.
